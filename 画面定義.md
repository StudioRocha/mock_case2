## 画面分類

### 一般ユーザー向け画面

-   PG01: 会員登録画面
-   PG02: ログイン画面
-   PG03: 勤怠登録画面
-   PG04: 勤怠一覧画面
-   PG05: 勤怠詳細画面
-   PG06: 申請一覧画面

### 管理者向け画面

-   PG07: ログイン画面
-   PG08: 勤怠一覧画面
-   PG09: 勤怠詳細画面
-   PG10: スタッフ一覧画面
-   PG11: スタッフ別勤怠一覧画面
-   PG12: 申請一覧画面
-   PG13: 修正申請承認画面

---

## 留意事項

### 動的セグメント

以下の画面では、URL パスに動的セグメント（パラメータ）が含まれます：

-   **PG05**: `/attendance/detail/{id}` - 勤怠レコードの ID
-   **PG09**: `/admin/attendance/{id}` - 勤怠レコードの ID
-   **PG11**: `/admin/attendance/staff/{id}` - スタッフ（ユーザー）の ID
-   **PG13**: `/stamp_correction_request/approve/{attendance_correct_request_id}` - 修正申請レコードの ID

### パスの重複

-   **PG06（一般ユーザー）**と**PG12（管理者）**は同じパス `/stamp_correction_request/list` を使用
-   認証ミドルウェアでユーザーのロール（`role`）を確認し、画面表示を切り替える
