# レスポンシブ対応確認方法（1400-1540px）

## 確認手順

### 1. ブラウザの開発者ツールを開く
- Chrome/Edge: `F12` または `Ctrl+Shift+I` (Windows) / `Cmd+Option+I` (Mac)
- Firefox: `F12` または `Ctrl+Shift+I` (Windows) / `Cmd+Option+I` (Mac)

### 2. レスポンシブデザインモードを有効化
- 開発者ツールの左上にあるデバイスアイコン（📱）をクリック
- または `Ctrl+Shift+M` (Windows) / `Cmd+Shift+M` (Mac)

### 3. 画面幅を1400-1540pxに設定
- 上部の幅の数値を直接入力して変更
- または、プリセットから「Responsive」を選択して手動で調整

### 4. 確認すべき画面とチェックポイント

#### 勤怠一覧画面 (`/attendance`)
- **URL**: http://localhost/attendance
- **チェックポイント**:
  - コンテナが中央に配置されているか
  - テーブルがはみ出していないか
  - パディングが適切か（左右50px）
  - テキストが切れていないか

#### 勤怠詳細画面 (`/attendance/{id}`)
- **URL**: http://localhost/attendance/{任意のID}
- **チェックポイント**:
  - コンテナが中央に配置されているか
  - グリッドレイアウト（120px 160px 150px 100px 150px）が崩れていないか
  - セクションのパディング（30px 50px）が適切か
  - 入力フィールドがはみ出していないか

#### 申請一覧画面 (`/stamp-correction-requests`)
- **URL**: http://localhost/stamp-correction-requests
- **チェックポイント**:
  - コンテナが中央に配置されているか
  - テーブルがはみ出していないか
  - タブが適切に表示されているか

#### 打刻画面 (`/attendance`)
- **URL**: http://localhost/attendance
- **チェックポイント**:
  - 中央配置が維持されているか
  - ボタンが適切なサイズで表示されているか
  - 時刻表示が切れていないか

### 5. 確認すべき幅
- **最小**: 1400px
- **最大**: 1540px
- **推奨**: 1400px, 1470px, 1540px の3点で確認

## 現在の実装状況

### ✅ 対応済み
- `common.css`: ヘッダー、フォームコンテナ

### ❌ 未対応
- `attendance/list.css`: 勤怠一覧画面
- `attendance/detail.css`: 勤怠詳細画面
- `stamp_correction_request/list.css`: 申請一覧画面
- `attendance/index.css`: 打刻画面

## 理論的な分析結果

### 各画面の設定
- **コンテナ幅**: `max-width: 1200px` (固定)
- **パディング**: `padding: 40px 50px` (固定)
- **実効幅**: 1200px + 100px (左右パディング) = 1300px

### 1400-1540pxでの動作
- 画面幅が1400px以上の場合、コンテナは中央に配置される
- 左右に余白が生じるが、レイアウト自体は崩れない可能性が高い
- ただし、**要件では「崩れないこと」が求められているため、メディアクエリでの調整が必要**

## 推奨対応
各画面のCSSに以下のメディアクエリを追加：

```css
@media (min-width: 1400px) and (max-width: 1540px) {
    /* 各画面のコンテナとパディングを調整 */
}
```
