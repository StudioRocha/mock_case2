# 共通化仕様

## 概要

勤怠管理アプリの共通化に関する仕様定義

---

## CSS 共通化

### CSS ファイル構成

```
resources/css/
├── app.css          # メインCSSファイル（エントリーポイント）
├── sanitize.css     # CSSリセット（sanitize.css）
└── common.css       # 共通スタイル（ヘッダー、ボタン、フォームなど）
```

### CSS インポート構造

`app.css`で以下の順序でインポート：

```css
/* CSSリセット */
@import 'sanitize.css';

/* 共通スタイル */
@import 'common.css';

/* その他のスタイル */
```

### 共通 CSS（common.css）の内容

以下の共通スタイルを定義：

1. **ヘッダー**

    - 黒背景（`#000000`）
    - 白文字（`#FFFFFF`）
    - ロゴ「CT COACHTECH」

2. **ボタン**

    - プライマリボタン（黒背景、白文字）
    - セカンダリボタン（必要に応じて）

3. **フォーム**

    - 入力フィールドのスタイル
    - ラベルと入力フィールドの配置

4. **リンク**

    - テキストリンクのスタイル

5. **レイアウト**
    - コンテナの最大幅
    - 中央揃え

## レイアウト共通化

### 共通レイアウトファイル

**ファイル**: `resources/views/layouts/app.blade.php`

**役割**: 全画面で共通するヘッダーやフッターを定義

**構造**:

```blade
<!DOCTYPE html>
<html>
<head>
    <!-- メタタグ、CSS読み込みなど -->
</head>
<body>
    @include('components.header')

    <main>
        @yield('content')
    </main>

    @include('components.footer') <!-- 必要に応じて -->
</body>
</html>
```

## コンポーネント化

### コンポーネントディレクトリ構造

```
resources/views/components/
├── header.blade.php      # ヘッダーコンポーネント
├── footer.blade.php      # フッターコンポーネント（必要に応じて）
├── button.blade.php      # ボタンコンポーネント
└── form/
    ├── input.blade.php   # 入力フィールドコンポーネント
    └── label.blade.php   # ラベルコンポーネント
```

### コンポーネント一覧

#### 1. ヘッダーコンポーネント（`header.blade.php`）

**用途**: 全画面で共通のヘッダー

**内容**:

-   黒背景のヘッダーバー
-   「CT COACHTECH」ロゴ

**使用方法**:

```blade
@include('components.header')
```

#### 2. ボタンコンポーネント（`button.blade.php`）

**用途**: 再利用可能なボタン

**パラメータ**:

-   `type`: ボタンの種類（primary, secondary など）
-   `text`: ボタンのテキスト
-   `url`: リンク先（オプション）

**使用方法**:

```blade
@include('components.button', [
    'type' => 'primary',
    'text' => '登録する',
    'url' => route('register')
])
```

#### 3. フォーム入力コンポーネント（`form/input.blade.php`）

**用途**: 再利用可能な入力フィールド

**パラメータ**:

-   `name`: フィールド名
-   `label`: ラベルテキスト
-   `type`: 入力タイプ（text, email, password など）
-   `value`: 初期値（オプション）
-   `required`: 必須フラグ（オプション）
-   `errors`: エラーメッセージ（オプション）

**使用方法**:

```blade
@include('components.form.input', [
    'name' => 'name',
    'label' => '名前',
    'type' => 'text',
    'required' => true
])
```

## 画面実装の流れ

### 1. 共通部分の実装

1. `resources/css/common.css`を作成
2. `resources/views/layouts/app.blade.php`を作成
3. `resources/views/components/header.blade.php`を作成
4. `resources/views/components/button.blade.php`を作成
5. `resources/views/components/form/input.blade.php`を作成

### 2. 各画面の実装

各画面では以下の手順で実装：

1. 共通レイアウトを継承
2. コンポーネントを使用
3. 画面固有の CSS を追加（必要に応じて）

**例: 会員登録画面**

```blade
@extends('layouts.app')

@section('content')
    <div class="register-container">
        <h1>会員登録</h1>

        <form method="POST" action="{{ route('register') }}">
            @csrf

            @include('components.form.input', [
                'name' => 'name',
                'label' => '名前',
                'type' => 'text',
                'required' => true
            ])

            <!-- 他のフィールド -->

            @include('components.button', [
                'type' => 'primary',
                'text' => '登録する'
            ])
        </form>

        <a href="{{ route('login') }}" class="login-link">ログインはこちら</a>
    </div>
@endsection
```

## デザイン仕様

### カラーパレット

-   **ヘッダー背景**: `#000000`（黒）
-   **ヘッダーテキスト**: `#FFFFFF`（白）
-   **ボタン背景**: `#000000`（黒）
-   **ボタンテキスト**: `#FFFFFF`（白）
-   **リンク**: `#0000FF`（青）または適切な色
-   **背景**: `#FFFFFF`（白）

### タイポグラフィ

-   **見出し**: 太字、適切なサイズ
-   **本文**: 読みやすいフォントサイズ
-   **フォント**: システムフォントまたは指定フォント

### レイアウト

-   **コンテナ最大幅**: 適切な幅（例: 600px）
-   **中央揃え**: `margin: 0 auto`
-   **余白**: 適切なパディングとマージン
