# integer と INT の違い

## 結論

**実質的には同じです**。Laravel のマイグレーションで`integer()`を使うと、MySQL では`INT`型として実装されます。

---

## 詳細説明

### 1. Laravel のマイグレーション（`integer()`）

Laravel のマイグレーションでは、整数型を定義する際に`integer()`メソッドを使用します。

```php
$table->integer('role');
$table->integer('status');
```

### 2. MySQL での実装（`INT`型）

Laravel の`integer()`メソッドは、MySQL では`INT`型として実装されます。

**実際の SQL**:

```sql
CREATE TABLE users (
    role INT NOT NULL,
    status INT NOT NULL
);
```

### 3. MySQL での`INT`と`INTEGER`の関係

MySQL では、`INT`と`INTEGER`は**完全に同じ**です。どちらも同じデータ型を表します。

```sql
-- これらは同じ
role INT
role INTEGER
```

MySQL の公式ドキュメントでも、`INT`と`INTEGER`は同義語として扱われています。

---

## 現在のプロジェクトでの使用状況

### マイグレーションファイルでの使用例

```php
// usersテーブル
$table->integer('role');

// attendancesテーブル
$table->integer('status');

// stamp_correction_requestsテーブル
$table->integer('status');
```

### 実際の MySQL での型

これらのマイグレーションを実行すると、MySQL では以下のように`INT`型として作成されます：

```sql
-- usersテーブル
role INT NOT NULL

-- attendancesテーブル
status INT NOT NULL

-- stamp_correction_requestsテーブル
status INT NOT NULL
```

## Laravel の整数型メソッド一覧

| Laravel メソッド    | MySQL 型       | 範囲                                                    | 説明                  |
| ------------------- | -------------- | ------------------------------------------------------- | --------------------- |
| `tinyInteger()`     | `TINYINT`      | -128 ～ 127                                             | 小さな整数            |
| `smallInteger()`    | `SMALLINT`     | -32,768 ～ 32,767                                       | 小さな整数            |
| `mediumInteger()`   | `MEDIUMINT`    | -8,388,608 ～ 8,388,607                                 | 中程度の整数          |
| `integer()`         | `INT`          | -2,147,483,648 ～ 2,147,483,647                         | 標準的な整数（32bit） |
| `bigInteger()`      | `BIGINT`       | -9,223,372,036,854,775,808 ～ 9,223,372,036,854,775,807 | 大きな整数（64bit）   |
| `unsignedInteger()` | `INT UNSIGNED` | 0 ～ 4,294,967,295                                      | 符号なし整数          |

---

## まとめ

1. **Laravel の`integer()`** = MySQL の`INT`型
2. **MySQL の`INT`** = MySQL の`INTEGER`（完全に同じ）
3. **現在のプロジェクト**では、Laravel の`integer()`を使用しており、MySQL では`INT`型として実装される
4. **設計案で「integer 型」と記載**されているのは、Laravel のマイグレーションでの表現方法を指している
