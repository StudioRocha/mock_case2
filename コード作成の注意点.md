# コード作成の注意点

## 概要

勤怠管理アプリのコード作成時に遵守すべき注意点一覧

---

## 注意点一覧

| 項目                                              | 要件                                                                                             | 詳細           |
| ------------------------------------------------- | ------------------------------------------------------------------------------------------------ | -------------- |
| インデント及び改行が整理できているか              | 不要な改行や不適切なインデントが存在しないこと                                                   |                |
| html のタグ構造が適当か                           | 見出しタグが h1 から順番に階層構造になっているか                                                 |                |
| html ファイルにおいて適切なタグ構成になっているか | div タグの乱用、li や table 等要素に合わせたタグ当て等ができているか                             |                |
| id 名と class 名の命名が適正か                    | id と class 名がローマ字など英単語でないものが使われていないか、意味の伝わる命名になっているか等 |                |
| 変数の命名が適正か                                | 意味のない命名をしない、ローマ字などの英単語ではないもので命名をしないなど                       |                |
| 存在するコメントアウトが適正か                    | 必要のないコメントアウトが残っていないか、コメントアウトが過剰になっていないかなど               |                |
| 各ファイル名の記法が適正か                        | 各ファイル名の記法が以下のルールを守っているか                                                   | 詳細は下記参照 |
| 機能要件の使用技術を遵守しているか                | 認証やバリデーションなど、指定した技術以外で実装されていないか                                   |                |
| クラスやファイルの読み込みは必要十分か            | 使用していないクラスやファイルを use で読み込んでいないか                                        |                |

---

## ファイル名の記法ルール

各ファイル名は以下のルールを遵守すること：

| ファイル種別               | 記法             | 例                                               |
| -------------------------- | ---------------- | ------------------------------------------------ |
| モデル名                   | アッパーキャメル | `User.php`, `Attendance.php`                     |
| コントローラー名           | アッパーキャメル | `AuthController.php`, `AttendanceController.php` |
| フォームリクエスト名       | アッパーキャメル | `RegisterRequest.php`, `LoginRequest.php`        |
| マイグレーションファイル名 | スネークケース   | `2024_01_01_000000_create_users_table.php`       |
| シーディングファイル名     | アッパーキャメル | `UserSeeder.php`, `AttendanceSeeder.php`         |

---

## 命名規則の詳細

### HTML/CSS 関連

#### id 名と class 名

-   ✅ **良い例**: `user-name`, `attendance-list`, `btn-submit`
-   ❌ **悪い例**: `userName`（キャメルケース）, `ユーザー名`（日本語）, `a1`（意味不明）

#### HTML タグ構造

-   見出しタグは`h1`から順番に階層構造にする
    -   ✅ **良い例**: `h1` → `h2` → `h3`
    -   ❌ **悪い例**: `h1` → `h3`（`h2`をスキップ）

#### 適切なタグの使用

-   `div`タグの乱用を避ける
-   リストには`<ul>`, `<ol>`, `<li>`を使用
-   テーブルには`<table>`, `<thead>`, `<tbody>`, `<tr>`, `<td>`を使用
-   セマンティックな HTML5 タグ（`<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>`など）を適切に使用

### 変数名

-   ✅ **良い例**: `$userName`, `$attendanceList`, `$isApproved`
-   ❌ **悪い例**: `$namae`（ローマ字）, `$a`（意味不明）, `$user_name`（スネークケースは PHP では非推奨）

### コメントアウト

-   必要のないコメントアウトは削除する
-   コメントアウトが過剰にならないようにする
-   残すコメントは、コードの意図を説明する有用なもののみ

## 技術要件の遵守

### 認証機能

-   **使用技術**: Fortify
-   他の認証ライブラリ（Laravel Breeze, Jetstream 等）は使用しない

### バリデーション機能

-   **使用技術**: FormRequest
-   コントローラー内での直接バリデーションは避ける

### メール認証（応用要件）

-   **使用技術**: Mailhog または Mailtrap
-   実装時は指定された技術を使用すること

---

## クラス・ファイルの読み込み

### use 文の管理

-   使用していないクラスやファイルを`use`で読み込まない
-   IDE の自動インポート機能を使用する場合は、未使用のインポートを削除する

### 例

```php
// ❌ 悪い例: 使用していないクラスを読み込んでいる
use Illuminate\Support\Facades\DB;
use App\Models\User;
use App\Models\Attendance; // このファイルでは使用していない

// ✅ 良い例: 必要なクラスのみ読み込む
use Illuminate\Support\Facades\DB;
use App\Models\User;
```

## チェックリスト

コード作成時は以下の項目を確認すること：

-   [ ] インデントと改行が整理されているか
-   [ ] HTML タグ構造が適切か（h1 から順番に階層構造）
-   [ ] 適切な HTML タグを使用しているか（div の乱用なし）
-   [ ] id 名と class 名が適正か（英単語、意味が伝わる）
-   [ ] 変数名が適正か（意味のある命名、英単語）
-   [ ] コメントアウトが適正か（不要なものは削除）
-   [ ] ファイル名の記法が適正か（モデル、コントローラー、フォームリクエスト、マイグレーション、シーディング）
-   [ ] 機能要件で指定された技術を使用しているか（Fortify、FormRequest 等）
-   [ ] 使用していないクラスやファイルを use で読み込んでいないか
